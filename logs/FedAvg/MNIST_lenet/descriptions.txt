  Train Loss  : Loss tren tap train
  Test Loss   : Loss tren tap test sach

------------------------------------------------------------------------
3.1  DBA - Distributed Backdoor Attack (ICLR 2020)
------------------------------------------------------------------------
Paper: Xie et al.
Loai : Data Poisoning - backdoor phan tan

Y TUONG:
  Thay vi cai cung 1 trigger (de bi phat hien), DBA chia trigger
  thanh 4 phan nho. Moi adversary chi cai 1 phan. Khi inference,
  ghep 4 phan thanh global trigger de kich hoat backdoor.

CACH HOAT DONG:
  1) Global trigger chia thanh 4 pixel pattern (8x1, gap=2):
       ****   ****
       (gap)
       ****   ****
  2) Adversary i cai local trigger (i mod 4):
     Adv 0: trai-tren, Adv 1: phai-tren,
     Adv 2: trai-duoi, Adv 3: phai-duoi
  3) Poisoning ratio = 32%
  4) Scaling factor = 100: scaled = G + 100*(X - G)
  5) Danh gia ASR: ghep ca 4 triggers tren anh test
  6) Chien luoc: continuous (tan cong moi epoch)
  7) Source->Target: 2->7 (all-to-one)

TAI SAO KHO PHONG THU:
  - Moi adversary chi cai 1 phan rat nho -> update gan benign
  - L2 distance giua adv va benign khong lon -> Krum/Bulyan kho loc
  - Scale factor bu cho trigger nho nhung TrimmedMean co the trim
  - Chi ghep du 4 phan moi co backdoor hoan chinh

CACH HOAT DONG:

3.3  Model Replacement - Constrain-and-Scale (AISTATS 2020)
------------------------------------------------------------------------
Paper: Bagdasaryan et al.
Loai : Data Poisoning + Model Poisoning

Y TUONG:

4.7  RFA - Geometric Median (IEEE TSP 2022)
------------------------------------------------------------------------
Paper: Pillutla et al.

NGUYEN LY:
  Thay trung binh cong bang trung vi hinh hoc (geometric median):
    v* = argmin_v Sum ||v - w_i||_2
  Dung Smoothed Weiszfeld (3 iterations):
    beta_i = alpha_i / max(||w_i - v||, eps)
    v = Sum(beta_i * w_i) / Sum(beta_i)
  Client xa (outlier) -> weight nho -> it anh huong.

UU DIEM:
  - Robust hon mean: xem xet TOAN BO vector (khong chi coordinate-wise)
  - Breakdown point = 1/2
  - Test Acc kha tot (96-97%)
  - Don gian, khong can aux data

NHUOC DIEM:
  - 3 iterations -> xap xi chua chinh xac
  - DBA adv update GAN benign -> geometric median bi anh huong
  - 8 adv (40%) -> gan breakdown point -> ASR=99.42%

THAM SO: num_iters=3, epsilon=1e-6

KET QUA:
  DBA, a=0.1, 4adv, 150ep : TestAcc=95.26%, ASR=11.13% -> TOT
  DBA, a=0.1, 8adv, 500ep : TestAcc=97.86%, ASR=99.42% -> THAT BAI
  BadNets, a=0.1, 4adv    : TestAcc=98.21%, ASR=96.63% -> THAT BAI


------------------------------------------------------------------------
4.8  FLAME - Taming Backdoors (USENIX Security 2022)
------------------------------------------------------------------------
Paper: Nguyen et al.

NGUYEN LY:
  3 LOP BAO VE:

  Lop 1 - Cosine Clustering:
    HDBSCAN(cosine distance, min_cluster=n/2+1=11)
    Cluster lon nhat = benign, loai bo con lai

  Lop 2 - Adaptive Clipping:
    S = median(||g_i||_2)
    Clip: g_i = g_i * min(1, S/||g_i||_2)
    Mean clipped gradients -> update global

  Lop 3 - Noise Injection:
    noise ~ N(0, (gamma*S*std(param))^2), gamma=1.2e-5
    Noise nhe du pha backdoor residual

UU DIEM:
  - 3 lop bo sung: clustering, clipping, noise
  - RAT HIEU QUA voi Model Replacement: clip loai scale lon
    -> ASR=10.61%, TestAcc=93.68%
  - Cosine distance khong bi danh lua boi scale

NHUOC DIEM:
  - HDBSCAN can min_cluster=n/2+1 -> adv < 50%
  - DBA (update gan benign): clustering co the khong tach duoc
  - Noise giam convergence: TestAcc thap hon Mean 1-3%
  - BadNets a=0.1: ASR=98.71% -> clustering that bai

THAM SO: gamma=1.2e-5

KET QUA:
  DBA, a=0.1, 8adv, 500ep  : TestAcc=96.14%, ASR=99.92% -> THAT BAI
  DBA, a=0.5, 4adv, 100ep  : TestAcc=93.35%, ASR=10.62% -> Tot
  ModelRepl, a=0.5, 4adv   : TestAcc=93.68%, ASR=10.61% -> TOT NHAT
  BadNets, a=0.1, 4adv     : TestAcc=95.95%, ASR=98.71% -> THAT BAI

4.9  FoolsGold - Sybil Detection (RAID 2020)
------------------------------------------------------------------------
Paper: Fung et al.

NGUYEN LY:
  1) Tich luy gradient history moi client qua cac epoch
  2) Normalize gradient (L2->1)
  3) Trich xuat indicative features: top-k (10%) trong output layer
  4) Tinh cosine similarity giua accumulated gradients
  5) Client giong nhieu client khac (cos_sim cao) -> giam weight (Sybil)
  6) Weight = logit(1 - max_cos_sim)

UU DIEM:
  - Chuyen phat hien Sybil (nhieu adv cung hanh vi)
  - Khong can biet so adversary

NHUOC DIEM:
  - Gia dinh adv GIONG NHAU -> DBA moi adv trigger KHAC -> khong phat hien
  - Non-IID: benign cung gradient khac nhau -> reweight SAI
  - THAT BAI HOAN TOAN: TestAcc=10.33% (random guess)

THAM SO: epsilon=1e-6, topk_ratio=0.1

KET QUA:
  DBA, a=0.5, 4adv, 100ep : TestAcc=10.33%, ASR=0.03% -> MODEL CHET


------------------------------------------------------------------------
4.10  FLARE - Latent Space PLR Defense (AsiaCCS 2022)
------------------------------------------------------------------------
Paper: Li et al.

NGUYEN LY:
  1) Extract PLR (Penultimate Layer Representation) tu moi client model
     bang forward hook tren last Linear layer
  2) Dung AUXILIARY DATA (test set, 1 class, 200 mau) cho forward pass
  3) Tinh MMD(PLR_i, PLR_j) giua moi cap client:
     MMD^2 = E[k(x,x')] + E[k(y,y')] - 2E[k(x,y)], kernel=RBF
  4) Nearest neighbor counting: dem client duoc chon la NN nhieu nhat
  5) Trust = exp(count / (tau * avg_count)), normalize -> trong so

UU DIEM:
  - Phat hien o LATENT SPACE -> manh hon parameter space
  - Adv model encode auxiliary data KHAC BIET -> PLR khac -> MMD lon
  - Weighted aggregation giu convergence

NHUOC DIEM:
  - CAN AUXILIARY DATA tu server -> gia dinh MANH trong FL thuc te
  - Chi dung 1 class -> khong phat hien tan cong class khac
  - 200 mau co the khong du dai dien

THAM SO: aux_data_num=200, selected_class=1, tau=1.0, mmd_kernel=rbf


------------------------------------------------------------------------
4.11  CPLRFL - Class-wise PLR for FL (PHUONG PHAP DE XUAT)
------------------------------------------------------------------------
Loai: Class-wise PLR-based trust scoring
KHONG CAN auxiliary data

NGUYEN LY:
  Cai tien FLARE: thay auxiliary data bang cross-client validation
  tren local data, phan tich THEO TUNG CLASS:

  Buoc 1 - Xac dinh classes moi client:
    Quet local dataset -> client i co classes nao
    Non-IID -> tap class khac nhau nhung CO OVERLAP
    VD: Client 0={0,1,3,7}, Client 1={1,2,5,7} -> overlap={1,7}

  Buoc 2 - Sample du lieu theo class:
    Moi class c: sample toi da 50 mau tu local data
    Tao dataloader rieng cho moi (client, class)

  Buoc 3 - Extract PLR theo class:
    Forward pass du lieu class c qua model client i -> PLR_{i,c}
    Forward hook tren last Linear layer (giong FLARE)

  Buoc 4 - Cross-client validation (CLASS-WISE):
    Voi moi class c, tim tat ca clients co class c
    Tinh MMD(PLR_{i,c}, PLR_{j,c}) cho moi cap (i,j) cung co class c
    Benign: PLR tren cung class TUONG TU (model hoc giong)
    Adversary: PLR tren TARGET CLASS khac biet (backdoor thay doi repr)

  Buoc 5 - Trust score:
    Thu thap tat ca MMD similarities moi client qua moi class
    Tinh MEDIAN similarity (robust voi outlier)
    Trust = exp(-median_sim / tau)
      Sim thap (gan client khac) -> score cao -> trusted
      Sim cao (xa client khac) -> score thap -> suspected
    Normalize trust scores tong = 1

  Buoc 6 - Weighted aggregation:
    w_global = Sum(trust_i * gradient_i)

SO SANH VOI FLARE:
  +----------------------------+-------------------+-----------------------+
  |         Tieu chi           |      FLARE        |       CPLRFL          |
  +----------------------------+-------------------+-----------------------+
  | Auxiliary data             | CAN (test set)    | KHONG CAN             |
  | Nguon PLR                 | Auxiliary data     | Local data client     |
  | Granularity               | 1 class duy nhat  | THEO TUNG CLASS       |
  | Similarity metric         | MMD->NN counting  | MMD->median->exp      |
  | Non-IID handling          | Khong thiet ke    | Tan dung class overlap|
  | Phat hien target class    | Chi selected_class| Moi class             |
  +----------------------------+-------------------+-----------------------+

UU DIEM:
  1) KHONG CAN AUXILIARY DATA -> thuc te hon trong FL
  2) CLASS-SPECIFIC detection: phat hien class nao bi poison
     DBA target class 7 -> PLR class 7 cua adv khac biet
  3) CROSS-CLIENT VALIDATION: tan dung class overlap non-IID
  4) Robust hon Mean/TrimmedMean voi tan cong latent space
  5) Weighted aggregation giu convergence

NHUOC DIEM:
  1) THOI GIAN: extract PLR moi (client x class) moi epoch
     -> cham hon Mean 10-20% (245ph vs 236ph cho 500ep)
  2) NON-IID QUA MANH (a=0.05): it class overlap -> trust kem
  3) ADVERSARY CAO (8/20=40%): majority assumption pha vo
     -> adv validate lan nhau -> trust score cao cho adv
     -> ASR=96.60% voi 8 adv
  4) DBA trigger rat nho -> PLR thay doi khong lon
  5) samples_per_class=50 co the khong du

THAM SO:
  samples_per_class=50, similarity_metric=mmd,
  tau=1.0, mmd_kernel=rbf, mmd_gamma=None

KET QUA:
  DBA, a=0.05, 4adv, 500ep : TestAcc=97.79%, ASR=10.78% -> TOT
  DBA, a=0.1, 4adv, 300ep  : TestAcc=97.34%, ASR=10.86% -> TOT
  DBA, a=0.1, 8adv, 500ep  : TestAcc=97.91%, ASR=96.60% -> THAT BAI
  BadNets, a=0.1, 4adv     : TestAcc=98.38%, ASR=97.20% -> THAT BAI
  ModelRepl, a=0.5, 4adv   : TestAcc=12.37%, ASR=58.69% -> THAT BAI
